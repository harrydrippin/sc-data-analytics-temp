# 4/10 실습

### 실습 목표

저장소의 `Practice/` 폴더 안에, **본인 이름으로 폴더를 만들고**, 그 안에 자기소개 파일 1개를 추가해서 — **PR을 만들고 Merge까지** 진행합니다.

### 실습 방법

모든 단계를 **Claude Code에게 시키면 됩니다.** 개념만 이해하고, 실행은 Claude에게 맡기세요. 커밋 메시지 형식이나 PR 구조는 `CLAUDE.md`에 정해져 있어서 Claude가 알아서 맞춰줍니다.

### 실습 단계

**1. 저장소 받기**

> 터미널에서 Claude에게: **“https://github.com/harrydrippin/sc-data-analytics-temp 클론해줘”**
> 
1. 이 때 어떤 폴더에서 생성하는지 확인해야 합니다. 보통은 Home 디렉토리에서 진행하며, 홈에서 클로드 실행 후 클론을 요청합니다. 

1. 홈에서 클론이 완료되어 폴더가 생성되었다면 클론을 종료하고 ls로 폴더 위치를 확인한 후에 cd sc-data-analytics-temp로 해당폴더로 이동하고 클로드를 실행합니다. 

⇒ 이미지처럼 나온다면 폴더 이동이 완료된 상태입니다. 여기서 클로드를 실행 후 나머지 작업을 이어하면 됩니다.

내 컴퓨터에 `sc-data-analytics-temp` 폴더가 생기면 완료입니다.

**2. 최신 버전 받기 (Pull)**

> Claude에게: **“main pull 받아줘”**
> 

`Practice/` 폴더가 보이면 완료입니다. 이것이 6-4에서 배운 Pull — “리모트의 최신 변경을 로컬로 받아오는 동작”의 실제 체험입니다.

**3. 브랜치 만들기**

> Claude에게: **“자기소개 파일 추가할 건데 브랜치 만들어줘”**
> 

어떤 작업을 할 건지 알려주면 Claude가 적절한 브랜치 이름(`본인이름/작업명` 형식)을 만들어줍니다. 본편(main)과 별개인 내 작업 공간이 생기면 완료입니다.

**4. 작업하기**

브랜치 안에서 자유롭게 작업합니다. `Practice/` 폴더 안에 본인 이름으로 폴더를 만들고, 자기소개 파일을 추가합니다.

```markdown
# 안녕하세요, 홍길동입니다
반갑습니다.
```

**5. PR 올리기**

> Claude에게: **“PR 올려줘”**
> 

Claude가 **커밋 + Push + PR 생성**을 한번에 처리합니다. 커밋 메시지와 PR 설명도 `CLAUDE.md` 규칙에 맞춰 자동으로 작성됩니다. GitHub에 PR이 생기면 완료입니다.

**6. 리뷰 & 수정**

GitHub 웹의 PR 페이지에서 세션 진행자가 리뷰를 달아드리겠습니다. e.g) 취미를 소개해주세요. 
이 내용을 반영해서 같은 브랜치에서 수정 작업을 합니다.

> 수정 후 Claude에게: **“push 해줘”**
> 

수정 사항이 PR에 자동으로 반영됩니다.

**7. Merge**

리뷰가 통과되면 GitHub 웹의 PR 페이지에서 **“Merge pull request”** → **“Confirm merge”** 를 클릭합니다. 본인의 폴더가 `main` 브랜치에 정식으로 들어가면 완료입니다.

### 완료 체크리스트

- [ ]  `main` 브랜치의 `Practice/` 폴더 안에 내 이름 폴더가 보인다
- [ ]  내 PR의 상태가 **Merged**(보라색)이다
- [ ]  저장소의 “Commits” 또는 “Insights → Contributors”에 내 이름이 나온다

세 개 모두 체크되었다면, 여러분은 **GitHub에서 협업하는 방법의 가장 핵심을 이미 익힌 것**입니다. 앞으로의 모든 GitHub 사용은 이 절차의 반복과 응용입니다.
